# Task: Create Architecture Decision Record

## Purpose
Document significant architectural decisions with context, alternatives, and consequences.

## Steps

1. **Determine ADR Number**
   - Check existing ADRs in `docs/ADR-*.md`
   - Use next sequential number

2. **Gather Context**
   - What problem are we solving?
   - What constraints exist?
   - What stakeholders are affected?

3. **Document Alternatives**
   - List at least 2-3 alternatives
   - Include "do nothing" if appropriate
   - Evaluate pros/cons of each

4. **Make Decision**
   - State the chosen approach clearly
   - Explain why this option was selected
   - Note any trade-offs accepted

5. **Document Consequences**
   - Positive outcomes expected
   - Negative impacts to manage
   - Neutral observations

6. **Create File**
   - Use template: `.bmad/templates/adr-template.md`
   - Save to: `docs/ADR-XXX-[slug].md`

## Template Location
`.bmad/templates/adr-template.md`

## Output
`docs/ADR-XXX-[descriptive-slug].md`
